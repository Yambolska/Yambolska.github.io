Loan Default Prediction — Credit Risk Modeling

Lending Club loan status prediction with ML models

Project Overview

This project uses a real-world Lending Club loan dataset to build a model that predicts whether a loan will be fully paid or defaulted. The workflow combines data analysis, feature engineering and model experimentation to obtain a reliable credit risk signal.

Data Analysis & Feature Engineering

  • Inspected missing values and applied selective imputation or removal
  • Identified highly correlated features and reduced multicollinearity
  • Dropped high-cardinality features with low predictive value
  • Converted loan_status into a binary target (Fully Paid vs. Charged Off)
  • Applied one-hot encoding to categorical features for model compatibility

Class Imbalance Handling

The target distribution is imbalanced: “Fully Paid” loans are much more frequent than “Charged Off”. Because of this, simple accuracy would be misleading, so the evaluation focuses on:

  • Precision
  • Recall
  • F1-score

This helps to better understand how well the model identifies truly risky loans instead of just predicting the majority class.

Models Used

  • Random Forest
  • Neural Network (Keras / TensorFlow)
  • XGBoost

Different models were trained and compared. XGBoost showed strong performance on non-linear relationships and sparse features, while Random Forest and the neural network provided useful baselines for benchmarking.

Key Insights & Outcomes

  • Loan grade (especially lower grades such as F and G) is a strong predictor of default risk.
  • The strong correlation between loan amount and installment guided feature selection.
  • Cleaning, encoding and targeted feature engineering had a clear impact on model quality.

The final setup provides a practical foundation for a credit risk scoring pipeline that can be adapted to fintech or lending products.
